TGIDE Image Detail Enhancement Project
Tornadogenesis-Inspired Image Detail Enhancement Algorithm (TGIDE)

=================================================
# Project Introduction
This project contains an image detail enhancement tool based on a swarm intelligence optimization algorithm (Tornadogenesis-Inspired Algorithm).
It works by decomposing the image into a base layer and a detail layer, utilizing the TGIDE algorithm to optimize high-frequency residuals, and finally re-fusing them to enhance the texture and edge details of the image.

# File Structure
```
├── TGIDE.m           # [Core Algorithm] Specific optimization and image processing functions
├── DEMO.m            # [Main Program] You need to save the provided second script as this filename (or main.m)
├── data/             # [Input Folder] Stores the .png images to be processed
├── result/           # [Output Folder] Automatically generated by the program, stores the enhanced images
├── Requirements.txt  # Environment dependency instructions
└── README.txt        # Usage instructions
```

# Quick Start
Step 1: Prepare the Environment
Ensure your MATLAB has the "Image Processing Toolbox" and "Parallel Computing Toolbox" installed.

Step 2: Prepare the Files
Save the contents of the first script as TGIDE.m.

Save the contents of the second script as DEMO.m.

In the directory where the code is located, create a new folder named data.
(If this folder does not exist, the program will not be able to find the images).

Step 3: Place Images
Place the .png images you want to enhance into the data folder.
Note: The code defaults to reading only the .png format. If you have .jpg images, please modify line 3 in DEMO.m.

Step 4: Run the Program
Open MATLAB and set the Current Folder to the directory containing the files above.

Type DEMO in the Command Window and press Enter, or simply click the "Run" button in the Editor.

# Parameter Description (can be modified in DEMO.m)
factor (Default: 4):
Enhancement intensity coefficient.

A higher value makes details more prominent but may introduce noise.

Recommended range: 2.0 - 5.0.

Pathr: Input image path (Default is '.\data').

Pathw: Output result save path (Default is '.\result').

# Notes
First-Run Speed: The program starts a parallel pool (parpool), so the first run might take a few dozen seconds to start the compute workers, which is normal. Subsequent image processing speeds will be much faster.

Memory Usage: For extremely high-resolution images (e.g., 4K and above), the Padding and population calculations in the TGIDE algorithm may consume a significant amount of memory.

Parallel Computing: TGIDE internally uses parfor to process image rows in parallel. If your computer has a high number of CPU cores, the processing speed will significantly improve.

# Output Results
The enhanced images will be saved in the result folder, with the filename format:
[Original Filename]_TGIDE_[Enhancement Intensity].png
For example: Lena_TGIDE_4.png
